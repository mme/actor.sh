# actor.sh V2

## Overview

V2 adds an optional Claude Code plugin on top of the existing skill and CLI. The plugin provides an MCP server with channel support, a status line showing actor state, and a local web dashboard for monitoring actors.

A single `pip install actor-sh` installs everything: the CLI, the MCP server, and all dependencies (mcp SDK, textual, textual-serve). The plugin is a thin wrapper that points Claude Code at the installed entry points.

```
actor package (commands.py, db.py, types.py, etc.)
  ├── CLI frontend      (cli.py — `actor` console script)
  ├── Skill frontend    (SKILL.md — tells Claude to shell out to `actor`)
  └── MCP frontend      (server.py — `actor-mcp` console script)
```

All three frontends share the same core Python package.

## Components

### MCP server (Python, stdio)

A Python MCP server using `modelcontextprotocol/python-sdk`, communicating with Claude Code over stdio.

**Tools exposed:**
- `run_actor` — spawn an actor in an isolated worktree
- `list_actors` — show all actors and their status
- `show_actor` — detailed info on a single actor
- `discard_actor` — stop and clean up an actor (cascading to children)
- `pr_actor` — create a PR from an actor's work

These are a new frontend to the existing `commands.py` logic.

**Channel (push notifications):**

The server declares the `claude/channel` experimental capability. When an actor's background process exits, the server pushes a `notifications/claude/channel` notification back into the Claude Code session with the result. No polling — the server manages the child processes directly and knows when they finish.

Example events:
- Actor finished successfully, with summary
- Actor failed, with error context
- Actor ready for PR

### Status line

A status line script bundled with the plugin. Displays running actor count, names, and status at the bottom of Claude Code. Includes an OSC 8 clickable link to the web dashboard at `localhost:<port>`.

### Web dashboard

Built with [Textual](https://textual.textualize.io/) and served in the browser via [textual-serve](https://github.com/Textualize/textual-serve). The dashboard is a pure Python Textual app accessible at `localhost:<port>`. The same app can also run directly in a terminal.

Runs in the same Python process as the MCP server (asyncio event loop handling both stdio and the textual-serve websocket server). Reads from the same SQLite database.

**Features:**
- List all actors with status
- View actor logs and output
- Diff viewer per actor using [textual-diff-view](https://github.com/batrachianai/textual-diff-view) — shows branch changes with syntax highlighting (unified or split view)
- Embedded terminal via [textual-terminal](https://github.com/mitosch/textual-terminal) for running commands in the context of an actor's worktree
- Clickable links to open/resume interactive Claude Code sessions for each actor

"Interact with an actor" means copying/clicking a link to its interactive session — not injecting messages into a running process.

## Installation

```
pip install actor-sh
```

This installs all dependencies (mcp SDK, textual, textual-serve) and two console scripts:
- `actor` — the CLI for human use
- `actor-mcp` — the MCP server entry point

## Plugin structure

A thin Claude Code plugin that points at the pip-installed entry points. No dependency management in the plugin itself.

```
actor-plugin/
  .claude-plugin/
    plugin.json
  skills/
    actor/
      SKILL.md          # existing skill (or updated version)
  agents/
    actor-sh.md         # custom agent with actor-aware system prompt
  .mcp.json             # points at `actor-mcp` console script
  settings.json         # { "statusLine": ... }
```

### Custom agent

The plugin ships a custom agent (`actor-sh`) that can be launched with `claude --agent actor-sh`. This agent has a system prompt tailored for orchestrating actors — it knows how to use the MCP tools, interpret channel notifications, and manage multi-actor workflows. Claude can also invoke it as a subagent when actor orchestration is needed during a regular session.

Installable via `/plugin install` from a marketplace, or `--plugin-dir` for local dev.

### Doctor command

`actor doctor` verifies the environment is set up correctly. Checks:
- Python version >= 3.9
- `actor-sh` package installed with all dependencies (mcp, textual, textual-serve)
- `actor` and `actor-mcp` console scripts on PATH
- SQLite database accessible at `~/.actor/actor.db`
- `claude` CLI installed and authenticated
- Plugin installed and enabled in Claude Code (if applicable)
- MCP server reachable
- Dashboard port available

Reports pass/fail per check with actionable fix suggestions.

## Technical notes

- The Python MCP SDK does not officially document channel support, but the mechanism is standard MCP notifications with method `notifications/claude/channel`. The SDK supports `server.send_notification()`, so this should work.
- The MCP server, web dashboard, and process manager all run in one Python asyncio process.
- The existing SQLite database (`~/.actor/actor.db`) and worktree layout (`~/.actor/worktrees/`) remain unchanged.
- The existing CLI and skill continue to work independently of the plugin — the plugin is purely additive.
